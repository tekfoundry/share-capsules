Share Capsules static host example

Two locally-created Capsules on one static page.

This example page shows the host-side markup for Capsules created by the development Creator extension. The encrypted .capsule files live beside this HTML page under ./capsules/.

Example 1

Logo card with contained media

A simple card keeps the whole logo visible by using a contained protected-content surface.

Code

<capsule-viewer
  src="./capsules/tekfoundry-logo.capsule"
  fit="contain"
  viewer-height="320px"
>
  <fallback>
    <p>Protected TekFoundry logo.</p>
  </fallback>

  <template>
    <figure class="result-shell subtle">
      <content class="viewer-frame logo" style="height: 320px;"></content>
      <figcaption>{{ title }}</figcaption>
    </figure>
  </template>
</capsule-viewer>

Result

Protected TekFoundry logo.

Example 2

Photo card with cover crop

A photo-oriented layout uses cover behavior so the Capsule feels like a normal responsive image card.

Code

<capsule-viewer
  src="./capsules/eclipse-photo.capsule"
  fit="cover"
  viewer-height="420px"
>
  <fallback>
    <p>Protected eclipse photo.</p>
  </fallback>

  <template>
    <article class="result-shell">
      <h3>{{ title }}</h3>
      <p>{{ description }}</p>
      <content class="viewer-frame photo" style="height: 420px;"></content>
    </article>
  </template>
</capsule-viewer>

Result

Protected eclipse photo.

Example 4

Contained photo with side filler

When the image should never crop or stretch, use contain. A portrait or tall photo stays centered and the empty space belongs to the frame.

Code

<capsule-viewer
  src="./capsules/eclipse-photo.capsule"
  fit="contain"
  viewer-height="320px"
>
  <fallback>
    <p>Protected eclipse photo.</p>
  </fallback>

  <template>
    <article class="result-shell subtle">
      <h3>{{ title }}</h3>
      <content
        class="viewer-frame photo-contain"
        style="height: 320px; background: #eef3fb;"
      ></content>
    </article>
  </template>
</capsule-viewer>

Result

Protected eclipse photo.

Example 6

Full-height image area

This pattern is useful when the design gives the Capsule a full-height area. The image height fills the viewing surface, stays centered, and can scroll horizontally when the natural width is wider than the area.

Code

  src="./capsules/eclipse-photo.capsule"
  fit="full-height"
  viewer-height="420px"
>
  <fallback>
    <p>Protected photo in a full-height area.</p>
  </fallback>

  <template>
    <article class="result-shell full-height">
      <h3>{{ title }}</h3>
      <content
        class="viewer-frame full-height"
        style="height: 420px; background: #f8fafc;"

    </article>
  </template>
</capsule-viewer>

Result

Protected photo in a full-height area.

Example 9

Custom safe error copy

The optional error section is hidden by default and is only for safe, user-facing messages when the Viewer cannot open the Capsule.

Code

<capsule-viewer
  src="./capsules/eclipse-photo.capsule"
  fit="contain"
  viewer-height="320px"
  debug
>
  <fallback>
    <p>Protected eclipse photo.</p>
  </fallback>

  <template>
    <content class="viewer-frame photo" style="height: 320px;"></content>
  </template>

  <error>
    <div class="error-card">
      <strong>Capsule unavailable</strong>
      <p>{{ error_message }}</p>
    </div>
  </error>
</capsule-viewer>

Result

Protected eclipse photo.

{{ error_message }}

Development note

These Capsules were created against the local development control plane and broker, so their manifests reference localhost:3003 and localhost:3004. They are useful for local static-host testing, not production viewing.

  • No creator recovery files are included.
  • No recovery code is included.
  • No source images are included.
