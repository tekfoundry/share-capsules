Share Capsules static host example

Static pages can host protected Capsules.

This example page shows the host-side markup for Capsules created by the development Creator extension. The encrypted .capsule files live beside this HTML page under ./capsules/.

View policy tour Open debug fixtures

Feature tour

One Capsule format, configurable access policies.

Time, Limit, and Trust policies can be used alone or combined. The Viewer opens the encrypted content only when every signed policy is satisfied.

Time Capsule

Opens only during the signed date window, such as after a launch date or before a campaign ends.

View time fixture

Limit Capsule

Limits how many times protected content can be viewed, either globally or per viewer account.

View limit fixture

Trust Capsule

Requires a viewer trust check using recent usage patterns and quick human challenges to reduce automated access.

View trust fixture

Combined Capsule

Uses multiple signed policies together. Content opens only when every configured rule is satisfied.

View combined fixture

Example 1

Logo card with contained media

A simple card keeps the whole logo visible by using a contained protected-content surface.

Code

<capsule-viewer
  src="./capsules/tekfoundry-logo.capsule"
  fit="contain"
  viewer-height="320px"
>
  <fallback>
    <p>Protected TekFoundry logo.</p>
    <a class="install-link" href="http://localhost:3003/viewer/install?return_to=http%3A%2F%2Flocalhost%3A8088%2F">
      Install the Viewer
    </a>
  </fallback>

  <template>
    <figure class="result-shell subtle">
      <content class="viewer-frame logo" style="height: 320px;"></content>
      <figcaption>{{ title }}</figcaption>
    </figure>
  </template>
</capsule-viewer>

Result

Protected TekFoundry logo.

Example 9

Custom safe error copy

The optional error section is hidden by default and is only for safe, user-facing messages when the Viewer cannot open the Capsule.

Code

<capsule-viewer
  src="./capsules/eclipse-photo.capsule"
  fit="contain"
  viewer-height="320px"
  debug
>
  <fallback>
    <p>Protected eclipse photo.</p>
    <a class="install-link" href="http://localhost:3003/viewer/install?return_to=http%3A%2F%2Flocalhost%3A8088%2F">
      Install the Viewer
    </a>
  </fallback>

  <template>
    <content class="viewer-frame photo" style="height: 320px;"></content>
  </template>

  <error>
    <div class="error-card">
      <strong>Capsule unavailable</strong>
      <p>{{ error_message }}</p>
    </div>
  </error>
</capsule-viewer>

Result

Protected eclipse photo.

Development note

These Capsules were created against the local development control plane and broker, so their manifests reference localhost:3003 and localhost:3004. They are useful for local static-host testing, not production viewing.

  • No creator recovery files are included.
  • No recovery code is included.
  • No source images are included.
