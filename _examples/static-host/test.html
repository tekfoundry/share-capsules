Bulk-page safety

Ordinary hidden layouts and queued openings

This section uses the same static Host contract in ordinary show/hide patterns: an open accordion, a closed details panel, and a tab or modal panel that starts hidden. The Viewer should discover explicit <capsule-viewer> elements in these layouts while opening only one Capsule pipeline at a time.

Hidden/show behavior: tabs, accordions, carousels, modals, and scripted reveal patterns are valid Host layouts.

Same-page queue: multiple eligible Capsules on this page wait for a shared Viewer opening slot before authorization, broker release, decryption, and rendering.

Retry-aware rate handling: temporary fetch, authorization, broker availability, and rate-limit failures keep a retry action available instead of replaying stale authorization material.

No surprise release consumption: connecting an account, approving disclosure, hidden markup, denied policy checks, revocation, and unredeemed tickets must not create a committed release. Only a successful broker key release counts as an opening.
