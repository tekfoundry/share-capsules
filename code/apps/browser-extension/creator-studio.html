    Hosting instructions

    Replace example.com with your real website when you publish.

    What your file host needs

    • Keep this exact Capsule revision at a permanent HTTPS URL.
    • Allow a public download without sign-in, cookies, or credentials.
    • Allow public cross-origin reads with Access-Control-Allow-Origin: *.
    • Serve it as application/octet-stream or application/zip. Supporting HEAD is recommended.
    • Keep useful public fallback content between the opening and closing <capsule-viewer> tags.

    Your host does not need a Share Capsules account, JavaScript SDK, database, key access, or server-side decryption. Test the page once with the extension disabled to confirm the fallback still makes sense.
