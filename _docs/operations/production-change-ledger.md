# Production Change Ledger

This ledger tracks the sanitized operational delta between the repository and the existing Share Capsules production environment. It is maintained as implementation progresses even when deployment is intentionally deferred.

Do not record passwords, tokens, private keys, connection strings, secret values, customer data, or other credentials here. Record environment-variable names and the required action only. Secret values belong in the production secret-management system.

## Deployment policy

- A completed development phase does not automatically trigger a production deployment.
- Deploy only after an explicit review confirms that the release contains a coherent, user-visible capability worth operating.
- Before deployment, reconcile this ledger with `code/.env.production.example`, migrations, scheduled commands, queue requirements, external identities, and infrastructure changes.
- Every deployed change requires verification and a rollback or forward-recovery decision.

## Existing production baseline

The production environment predates this ledger. Its sanitized configuration and operational baseline still need to be reconciled against the repository before the next production deployment.
