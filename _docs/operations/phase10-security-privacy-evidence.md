# Phase 10 Security, Privacy, and Compatibility Evidence Matrix

Status: Draft
Last updated: 2026-06-26

## Purpose

This matrix tracks the V1 release-security gates from the threat model and assigns each gate to evidence that can be audited before public release.

Evidence types:

- Automated test: PHPUnit, Vitest, Playwright, build, lint, scan, or reproducible command output committed or recorded with the release evidence.
- Operational check: a repeatable command, deployment check, repository setting, platform setting, or runbook exercise with recorded output.
- Manual review: documented human review with reviewer, date, scope, findings, and disposition.
- External review: independent security review with scope, date, reviewer, findings, severity, and remediation status.

Status legend:

- Complete: evidence exists and is linked here.
- In progress: implementation or partial evidence exists, but Phase 10 closure is incomplete.
- Open: no sufficient evidence has been recorded.

## Owners

| Owner | Responsibility |
|---|---|
| Engineering | Product code, automated tests, parser hardening, account lifecycle, extension behavior, and build artifacts. |
| Security | Threat-model evidence, abuse review, scan configuration, vulnerability handling, and security review coordination. |
| Operations | Deployment checks, backup/restore validation, monitoring, alerting, release credentials, and incident exercises. |
| Privacy | Consent, retention, export, correction, appeal, disclosure boundaries, and data-class review. |
| External reviewer | Independent review of cryptography, broker isolation, extension boundaries, parser hardening, and privacy lifecycle. |
