# Architecture Decision Records

Status: Draft
Last updated: 2026-06-18

This directory records consequential decisions that remain part of the current design intent. Design documents describe the current whole; decision records explain the rationale for current architectural constraints.

Superseded decision records should be removed or rewritten into the current canonical intent rather than retained as a history of abandoned designs.

Use sequential filenames such as `ADR-0001-trust-is-multidimensional.md`.

Each ADR should contain:

```markdown
# ADR-XXXX: Decision title

Status: Proposed | Accepted
Date: YYYY-MM-DD

## Context

## Decision

## Rationale

## Consequences

## Alternatives considered

## Related documents
```

Draft exploration belongs in the relevant design document or in `09_planning/open-questions.md`. An ADR should be added when the project deliberately commits to a consequential direction.
